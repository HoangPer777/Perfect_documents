# Technical Documentation Prompt

Sau khi hoàn thành công việc trong cuộc trò chuyện này, hãy tạo một file `technical_report.md` tổng hợp toàn bộ những gì đã thực hiện, theo cấu trúc sau:

---

## Yêu cầu nội dung file `technical_report.md`

### 1. Header
- Tên dự án / module
- giờ tạo - Ngày tạo / cập nhật

### 2. Tóm tắt thay đổi (Summary)
Danh sách ngắn gọn tất cả những gì đã làm, dạng bullet points.

### 3. Chi tiết từng thay đổi
Với MỖI thay đổi, cần trình bày đầy đủ:
- **Vấn đề / Mục tiêu:** Tại sao cần thay đổi này?
- **File đã chỉnh sửa:** Đường dẫn đầy đủ
- **Code trước và sau** (dùng diff hoặc code block so sánh)
- **Giải thích kỹ thuật:** Tại sao chọn cách này?

### 4. Cấu hình & Biến môi trường
Liệt kê tất cả:
- Biến môi trường cần thiết (`.env`)
- API Keys / Secrets (chỉ ghi tên biến, KHÔNG ghi giá trị thật)
- Cấu hình quan trọng (settings, config files)

Ví dụ format:
```
NEXT_PUBLIC_BACKEND_URL=http://localhost:8000   # URL Backend API
DATABASE_URL=postgresql://...                   # Chuỗi kết nối PostgreSQL
R2_ACCESS_KEY_ID=...                            # Cloudflare R2 Access Key
R2_SECRET_ACCESS_KEY=...                        # Cloudflare R2 Secret Key
```

### 5. API Endpoints
Với mỗi endpoint mới hoặc đã sửa:

| Method | Endpoint | Auth? | Request Body | Response | Mô tả |
|--------|----------|-------|-------------|----------|-------|
| GET | /api/v1/articles/ | No | - | `{count, results:[...]}` | Lấy danh sách bài báo |
| POST | /api/v1/articles/ | Yes (JWT) | `{title, slug, ...}` | Article object | Tạo bài báo mới |

### 6. Cấu trúc Database (nếu có thay đổi)
- Sơ đồ bảng (dạng text hoặc Mermaid ERD)
- Mô tả từng field mới/sửa
- Lệnh migration đã chạy

### 7. Data Flow / Sequence Diagram
Mô tả luồng dữ liệu chính bằng sơ đồ text (ASCII hoặc Mermaid):
```
Client → API → DB → Serializer → JSON Response → UI
```

### 8. Hướng dẫn cài đặt / Chạy lại
Các lệnh cần chạy để áp dụng thay đổi:
```bash
# Backend
pip install -r requirements.txt
python manage.py makemigrations
python manage.py migrate

# Frontend
npm install
npm run dev
```

### 9. Các vấn đề đã gặp & cách khắc phục (Troubleshooting)
Ghi lại các lỗi đã gặp và cách giải quyết để tránh lặp lại.

### 10. Các bước tiếp theo (Next Steps)
Những phần chưa làm hoặc cần cải thiện trong tương lai.

## Yêu cầu định dạng
- Viết bằng **Tiếng Việt**
- Code snippets phải có **syntax highlighting** (```python, ```tsx, ```bash...)
- Dùng **bảng Markdown** cho API endpoints và so sánh
- Dùng **Mermaid diagram** hoặc ASCII art cho sơ đồ
- File phải đủ chi tiết để một developer khác có thể hiểu và reproduce lại toàn bộ mà KHÔNG cần hỏi thêm
